**使用位运算改写**
在驱动 74HC138 译码器（控制位选）时，我们需要根据循环变量 `i` (0-7) 来控制三个引脚 `LSC, LSB, LSA` 的高低电平。

- **传统做法**：使用 `switch...case` 写 8 组判断，代码冗长。
    
- **进阶做法**：利用 `i` 的**二进制原形**，直接提取每一位的值赋给引脚。
    

### 2. 关键符号说明

- **`&` (按位与)**：
    
    - **规则**：全 1 为 1，有 0 则 0。
        
    - **用途**：作为“掩码”或“过滤器”。`i & 0x01` 表示只看 `i` 的最后一位，忽略前面所有位。
        
- **`>>` (右移)**：
    
    - **规则**：将二进制位向右移动指定的位数。
        
    - **用途**：把想要查看的那一位“拨”到最后一位，方便用 `&` 提取。
        

### 3. 代码逻辑详解（以 `i = 6` 为例）

十进制数字 `6` 在单片机内部存储为二进制：`1 1 0`。

| **代码逻辑**                     | **运算过程 (二进制)**                                      | **提取到的位**   | **赋值结果**    |
| ---------------------------- | --------------------------------------------------- | ----------- | ----------- |
| **`LSA = i & 0x01;`**        | `110 & 001` → `0`                                   | 第 0 位 (最低位) | **LSA = 0** |
| **`LSB = (i >> 1) & 0x01;`** | `110`右移1位变成`011`<br><br>  <br><br>`011 & 001` → `1` | 第 1 位 (中间位) | **LSB = 1** |
| **`LSC = (i >> 2) & 0x01;`** | `110`右移2位变成`001`<br><br>  <br><br>`001 & 001` → `1` | 第 2 位 (最高位) | **LSC = 1** |

**最终效果**：当循环到 `i=6` 时，ABC 引脚状态为 `110`，74HC138 自动选通第 7 位数码管（Y6）。

### 4. 优化后的驱动函数 (推荐写法)

C

```
void smg_display(void)
{
    u8 i;
    for(i=0; i<8; i++)
    {
        // 1. 消隐（关闭上一次的显示，防止重影）
        P0 = 0x00; 

        // 2. 位选：利用位运算自动拆分 i 的二进制位
        LSA = i & 0x01;        // 提取二进制第1位
        LSB = (i >> 1) & 0x01; // 提取二进制第2位
        LSC = (i >> 2) & 0x01; // 提取二进制第3位

        // 3. 段选：送出当前数字的段码
        P0 = gsmg_code[i]; 

        // 4. 延时：保持显示约 1ms
        delay_10us(100); 
    }
}
```

### 5. 易错点备忘录

1. **数据类型**：`i` 虽然定义为十进制，但在位运算时，单片机会自动处理其内存中的二进制形式。
    
2. **符号区分**：不要把按位与 `&` 写成逻辑与 `&&`（后者只用于 `if` 判断真假）。
    
3. **函数调用**：在 `main` 函数调用时必须带括号，如 `smg_display();`。
    
4. **消隐顺序**：先关闭段码输出，再切换位选，最后给新段码，显示效果最清晰。
