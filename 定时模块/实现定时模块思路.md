### 目标：
定时器计时模块开发，优化计时精度。实践内容为编写定时器中断计时程序，实现 1ms 精准计时，关联数码管显示更新。解决计时误差问题，确保每秒计数准确。

## 实现流程
**计时误差的消除（物理层）**

- **问题**：11.0592MHz 晶振下，1ms 对应 921.6 个机器周期，无法直接装载。
    
- **对策**：
    
    - **小数累加法**：通过 frac_acc 累加器，在 10 次中断中分配 4 次 921 和 6 次 922，从宏观上实现 1ms 的绝对平均值。
        
    - **中断响应补偿**：在计算初值时，预先扣除（或加上）中断入栈及指令执行消耗的 8 个机器周期，确保定时器计数的起点准确。
**显示抖动（Flicker）的消除（架构层）**

- **问题**：中断服务函数（ISR）执行时间波动（因除法运算）导致扫描频率不稳，以及位选切换时的重影。
    
- **对策**：
    
    - **任务解耦**：将耗时的“数据拆分/除法运算”移出中断，放入 main 循环。中断只负责“查表”和“IO翻转”，确保 ISR 执行时间恒定。
        
    - **消影（Ghosting）**：严格执行“关段选 -> 切换位选 -> 开段选”的顺序，彻底消除切换瞬间的残影。
        
    - **标志位通信**：使用 volatile 标志位实现中断与主循环的异步通信，保证数据更新的实时性。



### 📝 电子信息工程：定时器精准计时与显示优化笔记

#### 一、 精准计时：消除 11.0592MHz 晶振带来的累积误差

**1. 解决小数截断误差（小数累加法）**  
原理：1ms 需要 921.6 个机器周期，通过累加器在 10 次中断内分配 4 次 921 和 6 次 922，实现宏观上的零误差。

codeC

```
frac_acc += 6; // 每次累加 0.6 (放大10倍)
if (frac_acc >= 10) { frac_acc -= 10; /* 计 922 */ } 
else { /* 计 921 */ }
```

**2. 解决中断响应开销（软件补偿）**  
原理：单片机进入中断、压栈、执行赋值指令需要消耗约 8 个机器周期，必须在理论初值上加上这 8 个周期。

codeC

```
// 理论初值 64614 (0xFC66)，加上 8 个周期开销，实际赋 0xFC6E
TH0 = 0xFC; TL0 = 0x6E;
```

#### 二、 防抖动与防重影：显示架构优化

**3. 彻底消除重影（消影技术）**  
原理：在切换位选（74HC138）之前，必须先关闭段选（74HC245），防止上一位的残影带入下一位。

codeC

```
SMG_PORT = 0x00; // 1. 关段选 (消影)
// ... 切换位选逻辑 ...
SMG_PORT = smg_code[display_buf[digit]]; // 3. 送新数据
```

**4. 消除显示抖动（任务解耦）**  
原理：中断服务函数（ISR）必须执行时间恒定。将耗时的除法与取模运算移出中断，放入主循环，避免扫描频率波动。

codeC

```
// 中断内：仅设置标志位，不进行任何除法运算
if (ms_count >= 1000) { update_flag = 1; } 

// 主循环内：处理耗时的显存计算
if (update_flag == 1) { Update_Display_Buffer(total_seconds); update_flag = 0; }
```

**5. 异步通信机制（标志位）**  
原理：使用 volatile 关键字修饰标志位，确保主循环能实时读取到中断中修改的状态，防止编译器优化导致死循环。

codeC

```
volatile bit update_flag = 0; // 必须加 volatile，防止编译器缓存变量
```
